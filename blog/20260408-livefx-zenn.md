去年のzenn記事

### 環境整備

- gitlab→github移行
- eks→ecs以降
- タグ駆動ブランチングによってdev, sta, pro環境へのデプロイ
- githookを使ったbiomeフォーマットを強制
- dependabot追加とCI通過で自動マージaction追加
- discord botによる議事録作成

### リファクタ

- コントロールのサーバー統合（ `front-control-pc` と `back-control-pc` を `control-pc` へ統合）
- コントロールパネルの大幅刷新
- 全体的な重複コードやバグ修正

### 新機能

- NTP を利用して、端末間の時刻ずれを解消した演出
- インタラクティブな演出を追加（クライアントから桜を咲かせる）
- SNS 投稿機能

### ドキュメント

- notionのタスクを追加
- 重要なドキュメントはsharepointで共有

### AI

- discord botによる要約
- copilotを使ったPRレビューと実装
- 個人での実装はcursorやclaude codeなど

# 今年の追加事項(まだ途中 + 募集中)

## 1. 画面構成・遷移設計要件

### 1.1 トップページ / プロジェクト導線

- livefx.siw.ac.jp のトップに「プロジェクトページ」を配置
- プロジェクト一覧表示
    - 公開プロジェクト / 限定公開 / 非公開の区別
- 各プロジェクトへの遷移
- プロジェクト概要, 体験方法, 推奨環境の明示

### 1.2 認証・権限関連画面

- 認証導線の明確化
    - 未認証 → 認証 → プロジェクト利用
- 管理者用画面
    - ユーザー管理
    - プロジェクト管理
    - 環境(dev / sta)管理

## 2. 認証・アカウント基盤要件

### 2.1 認証方式

- Amazon Cognito 採用検討
    - 将来の移行を考慮した抽象化レイヤ設計
- 代替認証基盤の検討余地を確保
- Passkey(WebAuthn)対応検討
    - モバイル / App化時のUX考慮

### 2.2 権限管理

- ロールベースアクセス制御(RBAC: Role Based Access Control)
    - 管理者
    - 制作者
    - 一般ユーザー

### 2.3 管理画面

- 認証基盤導入前提で管理画面必須
- DB操作を伴う管理機能
    - ユーザー
    - プロジェクト
    - セッション

## 3. リアルタイム通信・セキュリティ要件

### 3.1 WebSocket挙動

- App(WebView含む)化を前提にした挙動再検証
- 再接続, タイムアウト, バックオフ制御
- モバイル回線劣化時の挙動確認

### 3.2 セキュリティ対策

- XSS(Cross Site Scripting)対策
    - CSP(Content Security Policy)設計
    - 入力値サニタイズ
- WebSocket経由データの検証
- トークン管理と失効戦略

## 4. 時刻同期・同期基盤要件

### 4.1 時刻同期方式

- AWS上にPodでNTPサーバを立てる案
    - ローカル + HWは性能不足の懸念あり
- WebSocket上にNTP的プロトコルを実装する案
    - RTT(Round Trip Time)補正
    - サーバ基準時刻配布

### 4.2 精度要件

- 演出・音響同期に耐えるレベルの時刻精度
- クライアント間同期誤差の測定と補正
- 超音波でSRから出してtime syncするとかも手段としてはある
    
    →リサーチ

## 5. 環境・運用要件

### 5.1 環境構成

- dev / sta / prod の明確な分離
- sta環境のコスト最適化
    - 作成から72時間経過後にAWSリソース自動削除
    - IaC(Infrastructure as Code)前提

### 5.2 CI/CD・通知

- GitHubイベント連動
    - PR, Merge, Deploy通知
- Discord Notifications連携
    - 開発・運用の即時可視化

## 6. Analytics・可観測性要件

### 6.1 Analytics導入

- 行動ログ取得
    - 画面遷移
    - インタラクション
- プライバシー・セキュリティ影響の事前確認
- 他機能への影響調査を前提に段階導入

### 6.2 ログ・モニタリング

- WebSocket接続状況
- 同期誤差
- エラー率

## 7. 演出・インタラクション機能要件

### 7.1 演出用インタラクション

- 端末を「振る」動作検知(なし/スマホが飛ぶので)
- 画面を「必死に触る」などの過剰入力検知
- マルチタッチ / ジェスチャー活用

### 7.2 音響・信号活用

- 音響信号による情報埋め込み
    - 可聴域外(超音波)
    - サウンドサイン
- 時刻同期・演出トリガとしての活用検討

### 7.3 参考事例
